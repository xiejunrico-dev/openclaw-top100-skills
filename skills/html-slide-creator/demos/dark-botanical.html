The Problem

Presentations shouldn't
require a build system

  • Complex toolchain: npm install, webpack config, 300MB of node_modules — before a single slide exists.
  • Generic output: Every AI deck looks the same — Inter font, purple gradients, predictable nothing.
  • No ownership: Closed platforms, export locks, monthly fees for work that should be yours.

Discovery

Style discovery that
actually works

You cannot describe design preferences in words. slide-creator generates three visual previews — you react, not describe. The "wow moment" arrives before a single line of content is written.

IAsk mood — Answer a few simple questions about tone and purpose
IIGenerate 3 mini previews — Three distinct visual directions, rendered instantly
IIIPick the one that feels right — Intuition over vocabulary

Workflow

From idea to presentation
in three unhurried steps

1/slide-creator — Answer four questions: purpose, length, content, images

3Open your presentation — Arrow keys · Space · Swipe · Click dots — navigation included

Features

Crafted for presentations
that command a room

Viewport fitting

Every slide fits exactly 100vh. No scrolling, no clipping, no compromise.

Inline editing

Press E, click any text, Ctrl+S to save. No code editor required.

PPTX export

/slide-creator --export pptx via Playwright and your own Chrome instance.

Two-stage workflow

--plan creates the outline first. --generate produces the final HTML.
